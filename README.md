# Vivi 老師｜建立我的 AI 小審

這是一份給非工程師使用的 AI 審查角色規格。你不需要從空白設計 Agent；Codex 會用一次只問一題的訪談，從你過去真正修改、退件或核准的成果中，整理出你的工作驗收標準。

## 直接開始

把下面整句複製到 Codex：

```text
請複製 Vivi 老師的 Git repo：https://github.com/vivi911/goaskvivi-ai-reviewer-starter，帶我完成小審的 AI Agent 流程。
```

Codex 複製 repo、讀取《建立我的AI小審.md》後，會直接問第一題，不需要再輸入一次「開始」。

如果你已經把 repo 放進 Codex 工作區，也可以直接輸入：

```text
開始
```

## 你會完成什麼

- 從真實工作案例找出通過標準與常見退件原因。
- 定義一票否決、完成證據、小審權限與退件後交接方式。
- 產出一份《我的工作驗收標準》。
- 用另一份過去真實成果做第一次驗證。
- 之後讓小審用 `Pass`、`Minor`、`Major` 或 `Blocked` 回報。

## 使用前請先遮蔽

不要提供密碼、驗證碼、Token 或登入畫面。材料若含姓名、電話、Email、訂單、客戶名單、帳號、未公開價格或公司內部數字，請先遮蔽或改成假資料。

## 重要邊界

- 小審預設只審不改，不會直接覆蓋原始檔。
- 看不到或讀不到的材料會標成「待確認」或 `Blocked`，不會腦補。
- 同一個對話切換角色比較接近 Persona；需要獨立審查時，請使用乾淨新對話或獨立子 Agent。
- 本 repo 不會替你登入、發布、發訊息、花錢或修改正式系統。

完整角色規格請見：[建立我的AI小審.md](./建立我的AI小審.md)。
